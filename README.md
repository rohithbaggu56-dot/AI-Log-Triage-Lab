# 🤖 AI Log Triage Lab – SOC Investigation Using AI Assistance

This project explores the use of an AI security assistant (Airia AI) to support SOC-style log analysis and alert triage workflows.

The goal was to understand how AI can assist analysts in interpreting logs, identifying suspicious behavior, and accelerating investigations.



## 🎯 Objective

- Use AI to analyze security logs
- Practice analyst-style investigation workflows
- Understand AI-assisted SOC operations
- Improve log interpretation efficiency

---

Logs generated from:

- Windows Event Logs
- Linux authentication logs
- Network activity simulations

Logs were analyzed using Airia AI for contextual explanation and investigation guidance.


## ⚙️ Activities Performed

- Submitted log samples to AI assistant
- Interpreted suspicious authentication events
- Identified anomaly patterns
- Performed investigation reasoning using AI explanations
- Compared manual vs AI-assisted analysis


## 🔎 Investigation Workflow

1. Review suspicious log event
2. Provide log data to AI agent
3. Analyze explanation provided
4. Validate indicators manually
5. Document findings


## 🧠 SOC Learning Outcome

This lab demonstrates how AI can assist — but not replace — security analysts by speeding up triage and improving understanding of complex logs.


## 🛠️ Tools Used

- Airia AI Agent
- Windows Logs
- Linux Logs
- SOC investigation methodology
